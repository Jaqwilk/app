**PRODUCT REQUIREMENTS (latest additions from user)**:
The user has requested a significant UI/UX overhaul for the home screen to include a series of sophisticated animations and interactions:
1.  **Animated Placeholder**: The placeholder text (What are you craving?) should have a typewriter animation, cycling through different short phrases in an infinite loop.
2.  **Keyboard-Aware Layout**: The Remaining calories pill and the + button at the bottom should move up when the keyboard is displayed to avoid being obscured.
3.  **Icon Change**: The + icon should be replaced with a more descriptive photo/image/file icon.
4.  **Complex Search Animation**: Instead of a simple loading state, a multi-stage animation sequence should play after the user submits a prompt:
    *   **Stage 1: searching**: A shimmer effect should pass over the word searching.
    *   **Stage 2: X sources**: The text changes to show a random number of sources (e.g., 9 sources) next to 3 overlapping circular icons. This text should also have a shimmer animation.
    *   **Stage 3: finalizing**: A final shimmer animation over the word finalizing.
5.  **Inline Meal Results**: After the animation, 3 meal options should slide out from under the user's prompt on the same screen, instead of navigating to a new page.
6.  **Detailed Recipe Page**: Tapping a meal result should navigate to a detailed recipe page. This page should display macros (calories, protein, carbs, fat) with small, distinctly colored dots next to each.
7.  **Animated Calorie Counter**: When a meal is logged and the user returns to the home screen, the number in the Remaining X cal pill should animate (e.g., scroll/count down) to the new value.
